git init

Initialize the current folder to a git env.

git clone

It doesn't really clone branches!

git checkout

cd for git

git checkout -b <staging>

touch <staging> + cd <staging> for git

git branch

Shows available branches
-a might help

git add <index.html>

Stages changes before commiting them

git commit

In VS code, a window will open asking for a commit message

git commit -m "commit message"

git commit with inline commit message

git push

Pushes to origin

git push --set -upstream origin <staging>

Pushes the <staging> branch to the repo for the first time

git remote add upstream git@github.com:<git username>/<repo>.git

Connects to the project owner repo

git pull <staging>

Clones <staging> branch (see above: clone doesn't do this!)
