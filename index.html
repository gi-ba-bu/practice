Git commands $ with sample code

git init

Sets up the current folder as a Git repository.

Used on either an empty folder or one already containing files.

git checkout -b <branch name>

Creates and names a new branch. Also makes the new branch the active one.

$ git checkout -b staging

git checkout <branch name>

Makes an existing branch the active one.

$ git checkout staging

git clone <repository>

Clones a repository; doesn't clone all branches.

Clones a github repository into a newly created directory with the same name, creates remote-tracking branches for each branch in the cloned repository (visible using git branch --remotes), but creates and checks out only one initial branch that is forked from the cloned repositoryâ€™s currently active branch.

$ git clone git@github.com:python/cpython.git

git branch

Shows available branches

git add <index.html>

Stages changes before commiting them

git commit

In VS code, a window will open asking for a commit message

git commit -m "commit message"

git commit with inline commit message

git push

Pushes to origin

git push --set -upstream origin <staging>

Pushes the currently active branch to the repository for the first time and names it

git remote add upstream git@github.com:<git username>/<repo>.git

Connects to the project owner repo

git pull <staging>

Clones <staging> branch (see above: clone doesn't do this!)
